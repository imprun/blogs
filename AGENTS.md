# 블로그 작성 가이드

> **AI Agent 필독**: 이 문서는 imprun.dev 블로그 작성 시 **반드시 따라야 할 강제 규칙**입니다.

## 핵심 원칙

**imprun.dev 기술 블로그는 전문 기업 기술 블로그 스타일을 따릅니다.**

- **회사 기술 블로그 스타일**: Netflix Tech Blog, Uber Engineering 수준의 전문적인 톤
- **객관적이고 기술적**: 감정적 표현이나 과장 없이 사실 중심
- **이모지 사용 자제**: 기본적으로 사용하지 않음
- **AI 티 제거**: "오늘은", "~해볼까요?", "~해봅시다" 등 금지
- **정확성 우선**: 추정치는 명확히 표기

**규칙 위반 시 즉시 재작성 요청됨.**

## 필수 블로그 구조

1. **메타데이터**: 작성일, 카테고리, 키워드
2. **요약**: 3-4문장 핵심 요약
3. **문제 상황**: 증상, 환경 구성, 초기 진단
4. **근본 원인 분석**: 기술 메커니즘, 설정 오류
5. **해결 과정**: 단계별 명령어와 검증
6. **재발 방지 방안 또는 교훈**
7. **참고 자료**: 관련 문서, 공식 문서 링크

## 블로그 발행 워크플로우

### 1. 블로그 파일 작성

`2025/MM/DD/` 폴더에 마크다운 파일 작성

### 2. README.md 업데이트

[README.md](./README.md) 테이블에 추가:

```markdown
| https://blog.imprun.dev/{다음번호} | [블로그 제목](2025/MM/DD/파일명.md) |
```

- **번호 규칙**: 테이블 최상단 번호 + 1
